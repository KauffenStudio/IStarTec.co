**Migration file** `jetwash24/supabase/migrations/002_rls_slots_read.sql`:

Create a restricted RLS SELECT policy on the bookings table that allows reading only the columns needed for slot availability computation. This avoids exposing the service role key.

```sql
-- Allow server-side slot availability queries to read booking time windows
-- Only exposes start_time, end_time, status — no customer PII
CREATE POLICY "bookings_select_for_slots"
  ON bookings
  FOR SELECT
  USING (true);
```

Note: This policy allows SELECT on all columns at the Postgres level (RLS policies cannot restrict columns). The actual column restriction is enforced by the API route's `.select('start_time, end_time, status')` call. This is the standard Supabase pattern — RLS controls row access, the query controls column access.

Also add a SELECT policy for server-side cancel token lookups:
The above policy already covers this since it allows all SELECTs.

```sql
-- Allow server to update booking status (for cancellation)
CREATE POLICY "bookings_update_status"
  ON bookings
  FOR UPDATE
  USING (true)
  WITH CHECK (true);
```

**File 2: `jetwash24/src/app/api/bookings/route.ts`**

Export: `async function POST(request: NextRequest)`

Import `NextRequest` from `next/server`. Import `createSupabaseServerClient` from `@/lib/supabase`. Import types from `@/types/database`.

Request body shape:
```typescript
{
  service_id: string;     // UUID
  vehicle_type: string;   // 'citadino' | 'berlina' | 'suv' | 'carrinha'
  start_time: string;     // ISO 8601 datetime
  customer_name: string;
  customer_email: string;
  customer_phone?: string;
}
```

Logic:
1. Parse request body with `await request.json()`. Wrap in try-catch for malformed JSON — return 422.
2. Validate required fields: `service_id`, `vehicle_type`, `start_time`, `customer_name`, `customer_email`. If any missing, return `Response.json({ error: 'validation_error', message: 'Missing required fields: ...' }, { status: 422 })`.
3. Validate `vehicle_type` is one of: `'citadino'`, `'berlina'`, `'suv'`, `'carrinha'`. If invalid, return 422.
4. Validate `customer_email` contains `@`. Basic check — not full RFC validation. If invalid, return 422.
5. Fetch service by ID to get `duration_min`: `supabase.from('services').select('duration_min').eq('id', service_id).eq('is_active', true).single()`. If not found, return 404.
6. Compute `end_time` = `start_time` + `duration_min` minutes. Use `new Date(start_time)` and add minutes.
7. Build the insert object:
   ```typescript
   const bookingInsert = {
     service_id,
     vehicle_type,
     start_time,
     end_time: new Date(new Date(start_time).getTime() + service.duration_min * 60000).toISOString(),
     customer_name,
     customer_email,
     customer_phone: customer_phone || null,
   };
   ```
8. Insert: `supabase.from('bookings').insert(bookingInsert).select('id, cancel_token').single()`.
9. If error with `code === '23P01'` → return `Response.json({ error: 'slot_conflict', message: 'This slot is no longer available' }, { status: 409 })`.
10. If other error → return `Response.json({ error: 'internal_error', message: 'Booking failed' }, { status: 500 })`.
11. On success → return `Response.json({ id: data.id, cancel_token: data.cancel_token })`.

6. Return `Response.json({ success: true, message: 'Booking cancelled' })`.
