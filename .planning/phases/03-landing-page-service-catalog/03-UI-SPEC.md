## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 48px (text-5xl) | 600 (semibold) | 1.1 | Hero headline only |
| Heading | 28px (text-3xl) | 600 (semibold) | 1.2 | Section titles (Services, Contact), service card names |
| Body | 16px (text-base) | 400 (regular) | 1.5 | Descriptions, about text, contact details |
| Label | 14px (text-sm) | 400 (regular) | 1.4 | Price labels, duration badges, tab labels, surcharge amounts |

Notes:
- Display and Heading share weight (semibold 600) — differentiated by size alone (48px vs 28px)
- Hero tagline "Detailing profissional, preços honestos." uses Display (48px, semibold)
- Service card names use Heading (28px, semibold)
- Label items that require visual emphasis (e.g. "Poupa Xé" badge text, struck-through prices) use Label (14px) with `line-through` or `font-semibold` utility — no additional weight introduced
- Two weights in use across the entire spec: 400 (regular) and 600 (semibold)

## Layout Contract

### Page structure
- Single scrollable page at `/[locale]`
- Section order: `<HeroSection>` → `<AboutSection>` → `<ServiceCatalog id="services">` → `<ContactSection id="contact">`
- No sidebar; single-column centered layout, max-width `max-w-6xl`, horizontally centered with `mx-auto`
- Page background: `bg-brand-navy` full bleed

### Navigation bar
- Sticky top nav: `position: sticky; top: 0; z-index: 50`
- Background: `bg-brand-navy/95` with `backdrop-blur-sm`
- Left: `<Logo variant="white" width={120} height={36} />`
- Right: anchor links to `#services` and `#contact` — 14px semibold, white, hover turns cyan
- Mobile (< 768px): hamburger button uses Lucide `Menu` icon with `aria-label="Abrir menu de navegação"` (PT) / `aria-label="Open navigation menu"` (EN) — reveals full-width dropdown nav
- No separate route — anchor scrolling only

### Hero section
- Full-bleed: `min-h-screen` or `min-h-[600px]` on desktop, `min-h-[480px]` on mobile
- Background: CSS gradient `from-brand-navy via-brand-navy to-brand-cyan-dark` (left-to-right or top-left to bottom-right diagonal)
- Content centered: `flex flex-col items-center justify-center text-center`
- Stack: Logo → Display tagline → CTA button → 3 badge icons
- Logo: `<Logo variant="white" width={200} height={60} />`, margin-bottom 32px
- Tagline: 48px semibold white, margin-bottom 16px
- CTA button: shadcn `<Button>` with `bg-brand-cyan text-brand-navy hover:bg-brand-cyan-dark`, `px-8 py-3`, links to `#services`
- Below CTA (margin-top 32px): 3 horizontal badges — Lucide icon + label — "Lavagem Interior", "Lavagem Exterior", "Pacotes" — white text, cyan icon

### Service catalog
- Vehicle selector: 4-button toggle group, full-width on mobile, inline on desktop — above the tab bar
- Selected vehicle button: `bg-brand-cyan text-brand-navy`; unselected: `bg-brand-navy-light text-brand-white border border-brand-cyan/30`
- Tab bar: shadcn `<Tabs>` with custom indicator — active tab underline in `#00C8E0`, 2px solid
- Service cards: 1-column on mobile, 2-column on tablet (768px+), 3-column on desktop (1024px+) grid
- Card background: `bg-brand-navy-light`, border: `border border-brand-cyan/20`, border-radius: `rounded-lg`

### Contact section
- Single panel on navy-light background, padding 48px vertical, 24px horizontal
- Content in 2-column grid on desktop (icon+label pairs), single column on mobile
- Each contact item: Lucide icon (16px, cyan) + text (16px, white)

## Interaction Contract

### Vehicle type selector
- Default state: Citadino selected on mount
- On select: updates `vehicleType` React state (useState in client component)
- All service card prices re-render immediately from props (surcharge logic: Citadino +0€, Berlina +5€, SUV +10€, Carrinha +15€)
- No page reload, no API call — client-side calculation from data loaded at server render

### Extras panel (interior card)
- Each extra: checkbox (`<input type="checkbox">`) with label showing name, duration addition, price addition
- Checked state is display-only in Phase 3 — no booking state yet; Phase 4 will consume selections
- Checking/unchecking updates a running "+total" display at bottom of card (e.g. "+65€ / +90min")

### Tab bar
- Default active: Interior
- Switching tabs filters the displayed service cards; hidden cards are not rendered (conditional render, not CSS display none)
- Tab switch is instant — no animation delay required, no loading state

### CTA button
- `href="#services"` with smooth scroll behavior (`scroll-behavior: smooth` on `<html>`)
- Phase 4 will change this to open booking flow — keep as anchor for now

### Hover states
- CTA button: `hover:bg-brand-cyan-dark` transition 150ms
- Nav anchor links: `hover:text-brand-cyan` transition 150ms
- Service cards: `hover:border-brand-cyan/50` transition 150ms — no lift/shadow effect

## Data Contract

Service catalog data source: Supabase `services` table (seeded via `seed.sql`). Server component fetches once on page load using `createSupabaseServerClient()`. Result passed as props to client components.

Vehicle surcharge logic (client-side, no additional fetch):

```
price = base_price_cents + surcharge_cents
surcharge: citadino=0, berlina=500, suv=1000, carrinha=1500 (cents)
display: Math.round(price / 100) + "€"
```

Package savings calculation (client-side):

```
saving = individual_total_cents - bundle_price_cents
display: "Poupa {{saving}}€" badge when saving > 0
crossed-out price: individual_total displayed with line-through
```
